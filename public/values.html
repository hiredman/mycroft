HomeValues

Values are data, whether simple (numbers, characters) or composite (vectors, maps, sets, etc.) All forms of composition nest to arbitrary depth, with no restrictions.

Values can be observed in a generic way: they are not concealed behind any class-specific API, nor do they require any coordination with other parties. (In particular, no locking!)

Values are immutable. They must be immutable, to allow generic observation.

Some value examples:

  • a set of maps
  • a heterogeneous collection
  • Clojure metadata
